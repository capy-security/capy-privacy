# Capy Privacy DNS

DNS filtering platform with DNS-over-HTTPS (DoH), a web UI, and an API to manage domains, clients, and blocklists.

## Architecture

```
                    Internet (clients)
                              │
                              ▼
    ┌─────────────────────────────────────────────────────────────┐
    │         Reverse proxy (Caddy · capy_front)                  │
    │              :80 / :443 · TLS (self-signed or Let's Encrypt)│
    └────────────────────┬────────────────────┬───────────────────┘
                         │                    │
           api.<domain>  │                    │  dns.<domain> / admin.<domain>
                         │                    │  · SPA + DoH
                         ▼ SPA calls API      │
                ┌───────────────┐             │
                │ capy_api      │             │
                │ FastAPI :8080 │             ▼
                │               │     ┌─────────────────┐
                └───────┬───────┘     │ capy_core       │
                        │             │ dnsdist         │
                        │             │ PowerDNS        │
                        ▼             └────────┬────────┘
                ┌────────────────┐             │
                │ database (vol) │             ▼
                └────────────────┘     ┌────────────────┐
                                       │ upstream       │
                                       │ resolvers      │
                                       └────────────────┘
```

**Components**

| Component                                 | Role                                                                                                   |
| ----------------------------------------- | ------------------------------------------------------------------------------------------------------ |
| **Reverse proxy** (Caddy in `capy_front`) | Publishes the API, DoH, and admin SPA. TLS termination, routing by host/path.                          |
| **Admin SPA**                             | Web UI at `admin.<domain>`; talks to the API. DoH at `dns.<domain>`.                                   |
| **capy_api**                              | REST API (domains, clients, categories, stats). Feeds blocklist/config to the core.                    |
| **capy_core**                             | DoH (`/dns-query` on :5300) and DoT (:853). dnsdist + PowerDNS Recursor; applies policy and recursion. |

## Requirements

- Docker and Docker Compose
- TLS certificates: **local** (self-signed, generated by setup script) or **internet** (Let's Encrypt via certbot)
- Host ports 80, 443 (front); optionally 53, 853 (core)

## Quick start (first time)

Run these steps from the **repository root** so `.env` is loaded by Docker Compose.

1. **First-time setup (creates `.env` and certificates)**

   ```bash
   ./certificates.sh
   ```

   (If that fails with "permission denied", run `bash certificates.sh`.)
   - Choose **1** for local (self-signed certs) or **2** for internet (Let's Encrypt).
   - For local: accept default domain `localhost` and IP `127.0.0.1` by pressing Enter, or type your LAN domain/IP.
   - The script writes `.env` (including `API_SECRET` for the API) and populates `resources/ssl/{api,dns,admin,ip}/` with the required certificates.

2. **Build and run**

   ```bash
   docker compose up -d --build
   ```

   From another directory, use: `docker compose --env-file /path/to/capy-privacy/.env -f /path/to/capy-privacy/docker-compose.yaml up -d --build`.

3. **Create the first admin**
   - Open the admin UI (e.g. `https://admin.localhost` for local; accept the self-signed cert in the browser).
   - Use the link **"Create admin account (first time only)"** under the sign-in form and create your admin user.
   - Then sign in with that account.

## Project layout

```
capy-privacy/
├── docker-compose.yaml   # API, core, front
├── prerequisites.sh      # First-time setup: .env (incl. API_SECRET) + SSL (self-signed or Let's Encrypt)
├── api/                  # FastAPI app, SQLite, domain/client/blocklist logic
├── core/                 # dnsdist + PowerDNS Recursor config
├── front/                # Caddy config, static admin SPA, blocked page
├── admin/                # SvelteKit source for the admin UI (built into front image)
└── resources/            # SSL, database, caddy custom config (created by setup)
```
